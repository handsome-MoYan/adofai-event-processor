1. 基本工作流程

详细步骤

  1. 选择输入文件：点击浏览…或将 .adofai / .txt 文件拖入窗口。
    • 支持格式：.adofai、.txt、.json
    • 自动检测编码（UTF-8、UTF-8-SIG、GBK、Shift-JIS、ASCII）
  2. 设置输出目录：点击自动填充使用输入文件所在目录，或浏览选择其他位置。
  3. 指定输出文件名：输入文件名（不含扩展名），留空则使用 原文件名_new.adofai。
  4. 添加筛选条件：使用关键词树添加主关键字或子关键字（支持正则表达式）。
    • 主关键字：主要搜索条件（同一分支内为"或"逻辑）
    • 子关键字：附加条件（与主关键字为"与"逻辑）
  5. 选择处理模式：从6种模式中选择（详见处理模式详解）。
  6. 开始处理：点击开始处理或按 Ctrl+Shift+P。
注意：如果输出文件已存在，工具会自动创建唯一文件名（如 文件名(1).adofai）。

3. 处理模式详解

模式说明使用场景
删除包含任意关键词的行 删除匹配至少一个主关键字的行（分支内"或"逻辑） 快速删除特定事件类型
提取包含任意关键词的行 仅保留匹配至少一个主关键字的行 分离特定事件用于分析
删除包含所有关键词的行 删除同时匹配所有主关键字及其子关键字的行 多条件精确删除
提取包含所有关键词的行 仅保留同时匹配所有主关键字及其子关键字的行 提取高度特定的事件
修改包含任意关键词的行 将匹配行的整行内容替换为指定文本 批量替换事件参数
替换匹配行中的子串 在匹配行中查找并替换文本（非整行替换） 更新特定值同时保留结构
重要：使用子关键字时，它们与所属主关键字形成"与"逻辑。不同主关键字之间形成"或"逻辑。

4. 工具箱功能

4.1 智能清理

自动清理 ADOFAI 文件中的常见问题：

空 MoveDecorations 清理器

删除引用不存在装饰物标签的 MoveDecorations 事件。

示例场景：
- 你有针对 "deco1" 的 MoveDecorations 事件
- 但不存在标签为 "deco1" 的装饰物
- 本工具会删除这些无用的 MoveDecorations 事件

未使用装饰物清理器

删除从未被其他事件引用的装饰物事件（AddDecoration、AddText 等）。

  • 分析来自 MoveDecorations、SetText、EmitParticle 等的引用
  • 处理 hitboxDecoTag 递归引用
  • 支持仅预览模式，先查看将被删除的内容
  • 修改前自动创建备份
最佳实践：始终先启用"仅预览"模式查看将被删除的内容，确认无误后再执行实际清理。

4.2 VFX制作

为视频录制创建优化版本的关卡：

模式用途删除内容
前景 / 谱子 用于游玩的谱面 背景装饰物、CustomBackground、HallOfMirrors
背景 / 录制 用于仅背景录制 游戏事件（SetPlanetRotation、ColorTrack 等）、前景装饰物

Video VFX Pro

基于深度值和遮罩设置的智能过滤：

  • 前景模式：保留负深度装饰物，根据遮罩设置过滤
  • 背景模式：保留非负深度装饰物
  • 智能处理 Flash 和 SetFilterAdvanced 事件
  • 尊重 maskingType 和 maskingDepth 设置

4.3 预设组

同时应用多个预设：

  1. 从下拉框选择预设组
  2. 选择追加模式（添加到现有）或替换模式（先清空）
  3. 预览显示所有子预设及其项目数量
  4. 所有关键字作为主串（顶层）添加

6. 使用预设

创建 presets.json

在应用程序目录创建名为 presets.json 的文件：

{
  "常用事件": {
    "旋转": ["SetSpeed", "Twirl"],
    "颜色": ["ColorTrack", "RecolorTrack", "AnimateTrack"]
  },
  "装饰物": {
    "所有装饰": ["AddDecoration", "AddText", "AddParticle", "AddObject"],
    "移动事件": ["MoveDecorations", "MoveTrack"]
  }
}

应用预设

  • 应用预设（主串）：将预设值添加为顶层关键字
  • 应用预设（子串）：将预设值添加为选中主关键字的子项
  • 应用预设组：在工具箱标签页批量应用整个组

8. 进阶技巧

正则表达式模式

使用正则模式进行复杂匹配：

示例模式：
"eventType":\s*"SetSpeed"     - 匹配 SetSpeed 事件（任意空格）
"floor":\s*(100|101|102)      - 匹配特定楼层号
"angle":\s*\d{3,}             - 匹配3位及以上数字的角度

性能优化

  • 极速模式：对大文件（>10MB）禁用进度更新，处理速度提升10倍
  • 批量处理：使用预设组在一次操作中应用多个筛选器
  • 编码选择：ADOFAI 文件使用 UTF-8-SIG 确保正确处理 BOM

处理大文件

  1. 文件大于10MB时启用极速模式
  2. 使用有限行数预览检查设置
  3. 如遇内存问题，考虑分块处理
  4. 启用"保存日志"跟踪处理历史

9. 常见问题

问题解决方案
"输入文件不是文本文件" 确保文件不是二进制格式。ADOFAI 文件应为 UTF-8 编码的文本。
处理时出现 UnicodeDecodeError 尝试其他编码（ADOFAI 用 UTF-8-SIG）或使用自动检测。
预览显示内容被截断 在预览设置中增加"最大预览行数/字符数"。
预设无法加载 检查 presets.json 是否为有效 JSON 且位于应用程序目录。
工具箱清理找不到装饰物 确保文件使用标准 ADOFAI 格式，包含 actions/decorations 数组。
大文件出现内存错误 启用极速模式或分小块处理。
主题无法应用 某些主题可能需要重启。尝试切换两次或重启应用。

10. 技术细节

文件结构

ADOFAI 文件采用 JSON Lines 格式：

{ "pathData": "...", "settings": {...} }
{ "floor": 1, "eventType": "SetSpeed", "speedType": "Bpm", "beatsPerMinute": 100 }
{ "floor": 2, "eventType": "Twirl" }
...

处理逻辑

  • 逐行处理：每行独立评估
  • 正则缓存：编译的模式会被缓存以提高性能
  • 内存高效：大文件使用流式处理
  • 线程安全：后台处理不阻塞界面

配置文件

文件用途
adofai_ep_config.json用户偏好设置
adofai_ep_history.json处理历史（最近10次操作）
presets.json自定义筛选预设
logs/调试日志文件（如启用）
