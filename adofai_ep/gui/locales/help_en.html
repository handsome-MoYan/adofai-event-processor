1. Basic Workflow

Step-by-Step Guide

  1. Select Input File: Click Browse… or drag a .adofai / .txt file into the window.
    • Supported formats: .adofai, .txt, .json
    • Auto-detects encoding (UTF-8, UTF-8-SIG, GBK, Shift-JIS, ASCII)
  2. Set Output Directory: Click Auto Fill to use the input directory, or browse to select another location.
  3. Specify Output Filename: Enter a filename (without extension), or leave blank for original_new.adofai.
  4. Add Filter Conditions: Use the keyword tree to add main keywords or sub-keywords (regex supported).
    • Main Keywords: Primary search conditions (OR logic within same branch)
    • Sub Keywords: Additional conditions (AND logic with parent)
  5. Select Processing Mode: Choose from 6 modes (see Processing Modes).
  6. Start Processing: Click Start Processing or press Ctrl+Shift+P.
Note: If the output file already exists, the tool automatically creates a unique filename (e.g., filename(1).adofai).

ModeDescriptionUse Case

Important: Sub-keywords form AND logic with their parent main keyword. Different main keywords form OR logic with each other.

4. Toolbox Features

4.1 Smart Cleanup

Automatically clean common issues in ADOFAI files:

Empty MoveDecorations Cleaner

Removes MoveDecorations events referencing non-existent decoration tags.

Example scenario:
- You have MoveDecorations events for "deco1"
- But no decoration exists with tag "deco1"
- This tool removes these useless MoveDecorations events

Unused Decoration Cleaner

Removes decoration events (AddDecoration, AddText, etc.) never referenced by other events.

  • Analyzes references from MoveDecorations, SetText, EmitParticle, etc.
  • Handles hitboxDecoTag recursive references
  • Supports dry-run mode to preview deletions
  • Auto-creates backups before modifications
Best Practice: Always enable "Dry Run Only" first to see what will be deleted, then execute actual cleanup after confirmation.

  • Background Mode: Keeps non-negative depth decorations

4.3 Preset Groups

  2. Choose append mode (add to existing) or replace mode (clear first)

  4. All keywords added as main strings (top level)

6. Using Presets

Creating presets.json

Create a file named presets.json in the application directory:

{
  "Common Events": {
    "Rotation": ["SetSpeed", "Twirl"],
    "Colors": ["ColorTrack", "RecolorTrack", "AnimateTrack"]
  },
  "Decorations": {
    "All Decorations": ["AddDecoration", "AddText", "AddParticle", "AddObject"],
    "Move Events": ["MoveDecorations", "MoveTrack"]
  }
}

Applying Presets

  • Apply Preset (Main): Add preset values as top-level keywords
  • Apply Preset (Sub): Add preset values as children of selected main keyword
  • Apply Preset Group: Batch apply entire groups in Toolbox tab

8. Advanced Tips

Regex Patterns

Use regex mode for complex matching:

Example patterns:
"eventType":\s*"SetSpeed"     - Match SetSpeed events (any whitespace)
"floor":\s*(100|101|102)      - Match specific floor numbers
"angle":\s*\d{3,}             - Match angles with 3+ digits

Performance Optimization

  • Turbo Mode: Disables progress updates for large files (>10MB), 10x speed boost
  • Batch Processing: Use preset groups to apply multiple filters in one operation
  • Encoding Selection: Use UTF-8-SIG for ADOFAI files to handle BOM correctly

Handling Large Files

  1. Enable Turbo Mode for files >10MB
  2. Use limited line preview to check settings
  3. If memory issues occur, consider chunk processing
  4. Enable "Save Log" to track processing history

9. Troubleshooting

IssueSolution
"Input file is not a text file" Ensure the file is not binary. ADOFAI files should be UTF-8 encoded text.
UnicodeDecodeError during processing Try a different encoding (ADOFAI uses UTF-8-SIG) or use auto-detection.
Preview shows truncated content Increase "Max Preview Lines/Chars" in preview settings.
Presets won't load Check that presets.json is valid JSON and located in the application directory.
Toolbox cleanup finds no decorations Ensure the file uses standard ADOFAI format with actions/decorations arrays.
Memory error with large files Enable Turbo Mode or process in smaller chunks.
Theme won't apply Some themes may require restart. Try switching twice or restart the app.

10. Technical Details

File Structure

ADOFAI files use JSON Lines format:

{ "pathData": "...", "settings": {...} }
{ "floor": 1, "eventType": "SetSpeed", "speedType": "Bpm", "beatsPerMinute": 100 }
{ "floor": 2, "eventType": "Twirl" }
...

Processing Logic

  • Line-by-line: Each line evaluated independently
  • Regex caching: Compiled patterns cached for performance
  • Memory efficient: Streaming processing for large files
  • Thread-safe: Background processing doesn't block UI
